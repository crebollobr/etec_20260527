O que é uma API?

API = Interface de Programação de Aplicações. É um “cardápio” de serviços que um sistema oferece para outro usar — sem precisar saber como ele funciona por dentro.

Seu app→ pede → API (garçom)→ busca → Servidor (cozinha)

Você pede pelo cardápio; o garçom (API) leva o pedido à cozinha e traz a resposta pronta. Você nunca entra na cozinha.

Como a conversa acontece

1. Requisição (GET)

O app pede dados a uma URL, usando um método HTTP (aqui, GET = “me dê”).

2. Resposta (JSON)

O servidor devolve os dados em JSON — um texto leve que o JavaScript entende facilmente.

E se a internet falhar?

    
Uma API pode estar fora do ar, ou o dado pode não existir. Por isso usamos try / catch: o app avisa o usuário em vez de “quebrar”.

    

      
Boa prática

      
Todo consumo de API deve prever o erro. Internet instável e respostas inesperadas fazem parte do mundo real.

Quem é você? E o que você pode fazer?

Objetivos

O que vamos aprender

  • Diferenciar autenticação de autorização
  • O fluxo login → token → acesso
  • O que é um token (introdução ao JWT)
  • Controle de acesso por papel (role) e boas práticas

Duas perguntas diferentes

🛂 Autenticação

“Quem é você?”
Confere suas credenciais (login e senha). É o check-in no aeroporto: mostrar o passaporte.

🎫 Autorização

“O que você pode fazer?”
Define a quais áreas você tem acesso. É o cartão de embarque: só o seu portão.

Primeiro o sistema descobre quem você é; depois decide o que você pode acessar.

O fluxo do login

Login + senha→ Servidor valida→ Emite um token→ Token acompanha as próximas requisições

O token é como um crachá digital: depois de autenticado, ele prova quem você é sem precisar reenviar a senha toda hora.
